3asyd

WebGL library tuned for Scientific Systems Modelling and Robotic Integration.

View project on GitHub

Reason for Existence

Over the last year or two I have really thought on the power of web technologies and have come to love them. I have noticed a distinct lack of tools for visualizing in 3D more technical systems, everywhere you look you can find libraries for 3D, but they really focus on the artistic and design side of things. I wanted to create something that could be used for remote vision systems or physics models and visualizing robotic sensor information. Things like that. I understand the speed gap that web technology faces when compared to more native tools such as c, c++ based graphics libraries, but I think the portability and ease of access adds a lot that you have to go through extra hoops with native languages to get. To note, I don't see this necessarily as an industrial applied software, but for DIY people a plug and play experience, built around the library and then of-course the accompanying library open to anyone who wants it.

Features

-Base systems for orthographic and perspective based 3d

-Mathematical functions that assist with graphics computations

-A few basic shapes (Cube,Sphere) plans to add a variety of shapes over time.

-Phong shading, flat shading.

-Object composition, add basic shapes to a mesh object, this mesh object acts as a single object at render time, allows handling of large amounts of objects (Tested about 100k cubes spinning on screen with above 30fps on a fairly average computer)
